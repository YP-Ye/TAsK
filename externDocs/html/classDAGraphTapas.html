Detailed Description

Implements main steps of TAPAS algorithm.

For details see [1]}.

Note
This implementation of TAPAS is not randomized, it does not implement flow proportionality condition. Also different condition is used in order to determine if PAS is inactive (PAS is inactive if flow was not moved during previous iteration). The following threshold is used in case of effective PAS creation where is current iteration number. Performance of the algorithm depends on this measure. Only creation of effective PASs is implemented. Brunch shift is not implemented.
